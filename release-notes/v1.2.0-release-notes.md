# Release Notes
## Version 1.2.0
Release Date: February 15, 2026

---

## Overview

Version 1.2.0 introduces improved validation for hazardous waste submissions, enhanced authentication security, and expanded reporting capabilities.

This release maintains backward compatibility with all existing v1 endpoints.

---

## New Features

### Bulk Report Submission
- Added support for batch submission of hazardous waste records.
- New endpoint:
  POST /reports/hazardous-waste/bulk
- Maximum batch size: 100 records per request.

### Audit Tracking Endpoint
- Added:
  GET /reports/{reportId}/audit
- Returns submission history and status changes.

---

## Improvements

- Improved validation messaging for missing required fields.
- Reduced response latency for report submission by ~15%.
- Enhanced OAuth token expiration error clarity.

---

## Bug Fixes

- Fixed issue where disposalDate validation incorrectly rejected ISO 8601 timestamps including timezone offsets.
- Corrected inconsistent error response format for 403 responses.

---

## Deprecations

None in this release.

---

## Upgrade Notes

No client-side changes required.

Clients using manual retry logic should review improved 401 and 403 error messaging for clearer troubleshooting.
