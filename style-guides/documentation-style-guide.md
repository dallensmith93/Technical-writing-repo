# Documentation Style Guide (Excerpt)

---

## Voice & Tone

- Clear and direct.
- Professional and neutral.
- Avoid unnecessary jargon.
- Write for the intended audience (developer vs stakeholder).

---

## Formatting Standards

- Use H2 (##) for major sections.
- Use H3 (###) for endpoint definitions.
- Use tables for field mapping and error codes.
- Use fenced code blocks for JSON examples.

---

## Terminology Consistency

- Use "endpoint" instead of "API call".
- Use "request payload" instead of "body" when describing JSON content.
- Use ISO 8601 when referencing date formats.

---

## Code Block Conventions

Always specify language when applicable:

```json
{
  "example": "value"
}
